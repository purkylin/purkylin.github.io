iptables

-t table: nat, filter
-s source
-A append to the chain
     INPUT：     位于 filter 表，匹配目的 IP 是本机的数据包 
     FORWARD：   位于 filter 表，匹配穿过本机的数据包， 
     PREROUTING： 位于 nat 表，用于修改目的地址（DNAT） 
     POSTROUTING：位于 nat 表，用于修改源地址 （SNAT
-D delete match from chain
-F delete all
-L list, 默认input
-s match source
-j DNAT 转换源IP
-j MASQUERADE 伪装IP，用于IP不固定的
--to-source 用于转换源地址(snat)，内网IP转换成公网IP，否则外网返回数据找不到合适的路由
  • eg1
    iptables -t nat -F PREROUTING
    清空 nat 表 PREROUTING 链中的所有规则
  • eg2
    iptables -A FORWARD -s 192.168.80.39 -j DROP
    阻止来源地址为 192.168.80.39 的数据包通过本机

  • eg3
    iptables -t nat -A POSTROUTING -s 192.168.0.0/24 -j SNAT –to-source 103.192.176.123

  • python 设置默认编码
    python的路径中，找到site-packages目录，在此目录中建立一个sitecustomize.py文件，设置相应的权限。文件的内容如下：
    import sys
    sys.setdefaultencoding(“utf-8”)
    或程序中加入
    #import sys
    #reload(sys)
    #sys.setdefaultencoding(‘utf-8’)
    注， reload 是必要的，python初始化之后就没有setdefaultencoding方法了

手动清理日志
/var/log/journal 存放着日志, rm 应该能工作. 或者使用journalctl,
例如:
清理日志使总大小小于 100M:

journalctl –vacuum-size=100M

清理最早两周前的日志.

journalctl –vacuum-time=2weeks

docker command

  • docker 常用命令

  • Dockerfile常用指令

    • FROM
    • ADD
    • RUN
    • COPY
    • EXPOSE
    • WORKDIR 设置工作目录
    • ENV 设置环境变量
    • ADD 复制文件自动解压，若尾字符是”/“,则复制文件夹
    • COPY 复制文件，不会解压
    • ONBUILD
    • USER 以哪个用记执行容器，默认root
    • VOLUME

  • 编写Dockerfile

  • 构建docker image
    docker build --no-cache -t purkylin/nginx
    尾部的.是必须的，代表当前目录，当然可根据需要修改， docker的每次构建都会cache， 若想禁用请加--no-cache参数

  • 查看docker虚拟机地址或虚拟机的ip地址，启动终端时会显示的, 通过命令行看不到，也可通客客户端工具看到
  • 暴露端口
    Dockerfile中暴露的端口号，在运行并不会暴露，只通执行run命令时制定，若不指定绑定胡宿主机器port，则随机; 也可通过-P选项来暴漏所有在Dockerfile中指定的port

  • 例子
    docker run –name nginx -d -p 8080:80 purkylin/nginx nginx -g “daemon off;”
    其中nginx后面的参数是必须的，让nginx在前台运行，否则nginx会立即结束，从而容器就停止运行
    绑定容器的80端口到宿主机器的8080端口上，不要弄反

  • volume 映射
    docker run --name nginx -d -p 80:80 -v $PWD/web:/usr/share/nginx/html/web purkylin/nginx nginx -g "daemon off;"
